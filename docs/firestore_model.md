# Firestore Data Model Documentation

This document describes the schema, field types, constraints, and collections designed for the Kisan Alert Firebase project.

---

## Collections Overview

The application utilizes four core collections in Firebase Firestore:
1. [farmers](#1-farmers) - Farmer profile and metadata.
2. [plots](#2-plots) - Plot geolocation and current crop metrics.
3. [village_defaults](#3-village_defaults) - Baseline soil nutrition data by village.
4. [escalations](#4-escalations) - Crop disease diagnosis escalation workflows.

---

## 1. farmers

Represents the profile of a registered farmer.

* **Collection Path:** `/farmers`
* **Document ID:** Auto-generated unique string ID.

### Schema

| Field Name | Type | Description | Default Value / Constraints |
| :--- | :--- | :--- | :--- |
| `phone` | String | Unique phone number used for WhatsApp communication. | Must be unique across all documents. |
| `name` | String | Display name of the farmer. | Required. |
| `village_id` | String | Identifier reference for the farmer's village. | Required. |
| `language` | String | Preferred communication language (ISO 639-1 code). | Default: `"te"` (Telugu). |
| `onboarding_stage` | String | Onboarding status of the farmer. | Default: `"new"`. |
| `created_at` | Timestamp | Server timestamp when the farmer registered. | Firestore Server Timestamp. |

---

## 2. plots

Represents a specific farm plot belonging to a farmer.

* **Collection Path:** `/plots`
* **Document ID:** Auto-generated unique string ID.

### Schema

| Field Name | Type | Description | Default Value / Constraints |
| :--- | :--- | :--- | :--- |
| `farmer_id` | String | The document ID of the associated farmer. | Foreign key to `farmers` collection. |
| `lat` | Number | Latitude of the plot's location. | Coordinate (float). |
| `lng` | Number | Longitude of the plot's location. | Coordinate (float). |
| `crop_current` | String | The crop currently sown in the plot. | Required. |
| `soil_data_ref` | String | Reference or identifier pointing to soil dataset. | Required. |
| `created_at` | Timestamp | Server timestamp when the plot was registered. | Firestore Server Timestamp. |

---

## 3. village_defaults

Stores standard default soil chemistry and nutrient metrics for a village. Used when a farmer does not have a physical Soil Health Card.

* **Collection Path:** `/village_defaults`
* **Document ID:** Custom string ID representing the unique village identifier (`village_id`).

### Schema

| Field Name | Type | Description | Default Value / Constraints |
| :--- | :--- | :--- | :--- |
| `n_kg_ha` | Number | Nitrogen content in kg per hectare. | Required (float/int). |
| `p_kg_ha` | Number | Phosphorus content in kg per hectare. | Required (float/int). |
| `k_kg_ha` | Number | Potassium content in kg per hectare. | Required (float/int). |
| `ph` | Number | Soil pH level (0-14). | Required (float). |
| `organic_carbon` | Number | Organic carbon percentage (%). | Required (float). |

---

## 4. escalations

Represents crop health alerts generated by Gemini Vision that require agricultural officer validation before dispatch to the farmer.

* **Collection Path:** `/escalations`
* **Document ID:** Auto-generated unique string ID.

### Schema

| Field Name | Type | Description | Default Value / Constraints |
| :--- | :--- | :--- | :--- |
| `plot_id` | String | Document ID of the plot with the issue. | Foreign key to `plots` collection. |
| `photo_url` | String | Public storage URL of the uploaded crop disease image. | Required. |
| `ai_diagnosis` | String | Crop disease/nutrient deficiency analysis by Gemini AI. | Required. |
| `status` | String | Validation state of the escalation. | Default: `"pending"`. Allowed: `["pending", "approved", "modified", "rejected"]`. |
| `officer_note` | String | Optional comments or annotations written by the officer. | Optional. |
| `final_message` | String | The actual message payload dispatched to the farmer. | Optional. |
| `created_at` | Timestamp | Server timestamp when the escalation was logged. | Firestore Server Timestamp. |
| `resolved_at` | Timestamp / Null | Server timestamp when the escalation was resolved. | Nullable. |

## Database Rules & Indexing

### Required Indexes
- Composite index on `/farmers` if querying both `phone` and other properties (e.g., sorting).
- Composite index on `/escalations` for query: `where("status", "==", "pending").orderBy("created_at", "desc")`.
